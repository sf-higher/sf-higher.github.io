1. 块级作用域绑定

1.1 提升(Hosting)机制

  • 在函数作用域中或全局作用域中通过关键字 var 声明的变量，无论在哪里声明，都会被当成在当前作用域声明的变量
  • 变量声明被提升至顶部，初始化操作则留在原处等待执行

1.2 块级声明

  • 块级声明用于声明在指定块的作用域之外无法访问的变量。块级作用域亦称词法作用域，存在于函数内部或块中（字符和{和}之间的区域）
1.2.1 let 声明
  • let 声明的用法与 var 相同
  • let 声明不会被提升
  • let 声明变量可以将变量的作用域限制在代码块中
  • 同一作用域中不能用 let 重复定义已经存在的标识符
  • 内部代码块中的同名变量会遮蔽全局作用域中的同名变量
1.2.2 const 声明
  • const 声明的是常量，其值一旦设定则不可更改，且使用 const 声明常量时必须进行初始化
  • 声明已经存在的标识符会造成语法错误
  • 如果 const 声明的常量是对象，允许修改值，但不允许修改绑定（修改地址）
1.2.3 let 与 const
  • let 与 const 声明的都是块级标识符，一旦代码执行到块外会立即被销毁

1.3 临时死区(Temporal Dead Zone)

  • JavaScript 引擎扫描代码时发现变量声明，var 声明会被提升至作用域顶部，其他的会放到临时死区中，执行变量声明语句后才移出
  • 访问临时死区中的变量会触发运行时错误
  • 在声明前访问由 let 定义的变量就会触发运行时错误

1.4 循环中的块级作用域绑定

  • let 和 const 声明在每次迭代循环都会创建一个变量，并以之前迭代中的同名变量的值将其初始化

    let 声明在循环内部的行为时标准中专门定义的，它不一定与 let 的不提升特性相关

1.5 全局作用域绑定

  • var 被作用于全局作用域，它会创建一个新的全局变量作为全局对象，这可能会覆盖掉一个已经存在的全局属性
  • let 与 const 在全局作用域下使用会创建一个新的绑定，只会遮蔽全局属性，不会覆盖

1.6 块级绑定最佳实践

  • 默认使用 const， 只在确实需要改变变量的值时使用 let

2. 字符串和正则表达式

2.1 更好的Unicode支持

2.1.1 UTF-16码位
  • 在UTF-16中，前2^16^个码位均已16位的编码单元表示，这个范围被称为基本多文种平面(BMP).超出这个范围的码位则要归于某个辅助平面
  • 字符串里的字符有两种，一种是由一个编码单元16位标识的BMP字符，另一种是有两个编码单元32位标识的辅助平面字符
  • 在ECMAScript5中，所有字符串的操作都基于16位编码单元

2.1.2 codePoint()

  • ECMAScript6新增
  • 接收编码单元的位置作为参数，返回域字符串中给定位置对应的码位，即一个整数值
  • 对于BMP字符集中字符，codePoint() 方法的返回值域 charCodeAt() 方法相同，对于非 BMP字符集则不同
  • 检测一个字符占用的编码单元数量的最简单的方法是codePointAt()

2.1.2 String.fromCodePoint()

  • 作用：
    • 根据指定的码位生成一个字符
  • 可以将其看成更完整版的String.fromCharCode().同时这两个方法对于BMP字符集的执行结果相同，非BMP字符集则有可能不同

2.1.3 normalize()

  • 作用：
    • 提供Unicode的标准化形式
  • 接收一个可选的字符串参数，指明应用某种Unicode标准化形式
  • Unicode标准化形式
    • 以标准等价方式分解，然后以标准等价方式重组(“NFC”),默认选项
    • 以标准等价方式分解(“NFD”)
    • 以兼容方式分解(“NFKC”)
    • 以兼容等价方式分解，然后以标准等价方式重组(“NFKD”)
  • 注意：在对比字符串之前，一定先把它们标准化位同一种形式

2.1.4 正则表达式 u 修饰符

  • 作用：
    • 当一个正则表达式添加了 u 修饰符，它就从编码单元操作模式切换为字符模式
    • 计算码位数量（效率低）
  • 检测 u 修饰符支持最安全的方式
  • 注意：
    • 如果代码仍然需要运行在老师JavaScript引擎中，那么使用 u 修饰符切记使用RegExp构造函数，可以避免发生语法错误

2.1.5 字符串中的字串识别

  • includes()
    • 在字符串中检测到指定文本返回true，否则返回false
  • startsWith()
    • 在字符串起始位置检测到指定文本返回true，否则返回false
  • endsWith()
    • 在字符串结束部分检测到指定文本返回true，否则返回false
  • 以上方法接收接收两个参数：(搜索的文本,[开始搜索的位置的索引])
    • 指定了第二个参数，前两个方法从索引处开始匹配，endWith()则从索引处减去欲搜索的文本长度的位置开始正向匹配
    • 不指定第二个参数，前两个方法从字符串起始处开始匹配，endWith()从末尾处开始匹配
  • 注意：
    • 如果使用正则表达式作为第一个参数，会发生错误
  • repeat()
    • 作用：
      • 返回当前字符串重复一定次数后的新字符串
    • 接受一个number类型参数

2.1.6 y 修饰符（粘滞修饰符）

  • 作用：
    • 影响正则表达式搜索过程中的sticky属性，挡在字符串中开始字符匹配时，他会通知搜索从正则表达式的lastIndex属性开始进行，如果在指定位置没能成功匹配，则停止继续匹配
  • 注意：
    • 只有调用 exec() 和 test() 这些正则表达式对象时才会设计lastIndex属性
    • 如果使用’^’来匹配字符串开端，只会从字符串的其实位置或多行模式的首行进行匹配
  • 如果JavaScript引擎支持奶奶之修饰符，则sticky属性的值为true,否则为false，且这个属性是只读

2.1.7 正则表达式的复制

  • 第一个参数为正则表达式，第二个为修饰符。使用第二个参数则会修改新生成的正则表达式的修饰符

2.1.8 flags 属性

  • 只读的原型属性访问器，只包含了getter方法
  • 扩展：
    • source
      • 返回正则表达式的内容
  • 作用：
    • 返回所有应用于当前正则表达式的修饰字符串

2.1.9 模板字面量

  • 基础语法： let message = \Hello World` `
  • 多行字符串
  • 换行
  • 字符串占位符
    • ${}
    • 可以将变量嵌入字符串中
  • 注意：
    • 在反撇号中，所有空白都属于字符串的一部分
